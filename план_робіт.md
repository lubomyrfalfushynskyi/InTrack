## 1. ФАЗА I — ПРАВКИ v2.2 ← РОБИТИ НАСТУПНИМ

### I1. Schema
- `asset_types`: + `normative_hours INTEGER` (норматив напрацювання, годин).
- Нова таблиця **`asset_usage`** (журнал напрацювання): `usage_id` PK, `asset_id` FK,
  `period_year`, `period_month` (1–12), `hours` (NUMERIC), `entered_by` FK users,
  `entered_at`. UNIQUE(asset_id, period_year, period_month).

### I2. Бекенд — терміни (дні + години)
- Залишковий строк за днями (наявне) + залишок за годинами = `normative_hours − SUM(usage.hours)`.
- Ефективний стан:
  • written_off/transferred — як є;
  • по днях ≤ 0 → «Вичерпано термін: −N днів» (доступне «Продовжити»);
  • по годинах ≤ 0 → ЧЕРВОНА підсвітка рядка БЕ тексту (акта не вимагає);
  • інакше «В експлуатації».
- Продовження: +1 рік; норматив годин НЕ зростає (нічого не робить із годинами).
- Новий маршрут `/api/assets/:id/usage` (GET — журнал; POST — внести місяць; PUT/DELETE — адмін).
- Розширити computed-SELECTassets: додати `remaining_hours`, `usage_total`, `expired_by_days`,
  `expired_by_hours`.

### I3. Бекенд — акти й валідація
- Передача ВСЕРЕДИНІ підрозділу: `from===to` → не списує, не міняє статус; міняє
  утримувача + (опц.) приміщення (за замов. — поточне). Передача МІЖ підрозділами:
  статус `transferred` у відправнику (наявне).
- Валідація дат: «остання дія» = max(action_date) по актах об'єкта.
  • Введення: action_date ≤ act_date.
  • Перенесення/Продовження/Списання: action_date ≥ остання дія; action_date ≤ act_date.
  → 400 з повідомленням при порушенні.
- Створення: підрозділ авто = підрозділ користувача (для scoped); глобальний адмін
  обирає цільового користувача → підрозділ з нього (заборонити інший).
- Видалення об'єктів: дозволити адміну підрозділу (у межах доступу), не лише глобальному.

### I4. Фронтенд
- Таблиця майна: **ресайз колонок** + **ширина по вікну** (AntD Table `resize` через
  `react-resizable` або `components.header.cell`; `scroll={{ x: 'max-content' }}`);
  сортування за вказаними колонками; за замов. — Найменування; оптимізовані ширини.
- Нотатки прибрати з таблиці; лишити лише у картці.
- «Додатково» → повна розшифровка без абревіатури «МОЛ».
- Стан: «В експлуатації»/«Вичерпано термін: −N днів»; червона підсвітка по годинах без тексту.
- Картка майна: блок «Журнал напрацювання» (таблиця 12×N місяців, внесення, хто вніс);
  кнопка «Продовжити» лише коли вичерпано по днях.
- Акти: форми введення з вибором одержувача → авто підрозділ; валідація дат (підсвітка/текст).
- Приміщення: дерева-блоки для 6 областей (Київська, Харківська, Полтавська, Рівненська,
  Одеська, Житомирська; фікс. порядок); решта алфавіт; підрозділи в колонку, будівлі в рядок.
- Інфо-кнопка на Решті вкладок (Приміщення, Користувачі, Види, Журнал).

**DoD I:** об'єкт з нормативом 5р/40000год; вичерпання по днях → «Продовжити» доступне,
+1 рік; вичерпання по годинах → червоний рядок без тексту; журнал напрацювання вноситься
щомісяця з фіксацією автора; передача всередині підрозділу міняє утримувача без списання;
валідація дат блокує некоректні; таблиці ресайзяться і по ширині вікна.

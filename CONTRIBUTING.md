# Contributing to Asset Management System

Дякуємо за ваш інтерес до внеску вкладу в розвиток системи обліку майна!

## Як зробити внесок

### Вимоги

- Node.js 18+
- Docker 20.10+
- Docker Compose 2.0+
- Git

### Процес розробки

1. **Fork проекту**
   ```bash
   # Натисніть кнопку "Fork" на GitHub
   ```

2. **Клонуйте ваш fork**
   ```bash
   git clone https://github.com/YOUR_USERNAME/asset-management-system.git
   cd asset-management-system
   ```

3. **Створіть feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Зробіть зміни**
   - Дотримуйтесь існуючого стилю коду
   - Додавайте коментарі до складного коду
   - Оновлюйте документацію при необхідності

5. **Тестуйте зміни**
   ```bash
   # Запустіть систему локально
   docker-compose up -d

   # Перевірте, що все працює
   # http://localhost:3000
   ```

6. **Комітайте зміни**
   ```bash
   git add .
   git commit -m "Опис ваших змін"
   ```

7. **Push в ваш fork**
   ```bash
   git push origin feature/your-feature-name
   ```

8. **Створіть Pull Request**
   - Перейдіть на GitHub
   - Натисніть "New Pull Request"
   - Додайте опис змін
   - Чекайте на рецензію

## Коди стилів

### JavaScript/React

- Використовуйте функціональні компоненти та hooks
- Дотримуйтесь принципів чистого коду
- Використовуйте meaningful names для змінних та функцій

### Backend (Node.js)

- Використовуйте async/await замість callbacks
- Додавайте обробку помилок
- Використовуйте meaningful names для API endpoints

### SQL

- Використовуйте uppercase для SQL ключових слів
- Додавайте коментарі до складних запитів
- Використовуйте індекси для оптимізації

## Звіт про проблеми

Якщо ви знайшли проблему:

1. Перевірте, чи існує вже issue
2. Створіть новий issue з описом проблеми
3. Додайте:
   - Очікувану поведінка
   - Фактична поведінка
   - Кроки для відтворення
   - Скріншоти (якщо можливо)
   - Версія системи

## Питання та пропозиції

Для питань та пропозицій:
- Створіть issue з тегом "question"
- Або зв'яжіться з командою через email

## Ліцензія

Вносячи зміни, ви погоджуєтесь, що ваш внесок буде ліцензовано під MIT License.

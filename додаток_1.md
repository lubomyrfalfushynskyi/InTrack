## 3. АРХІТЕКТУРА БД

### **3.1. Таблиця `regions` (області)**
```sql
CREATE TABLE regions (
  region_id SERIAL PRIMARY KEY,
  region_name VARCHAR(255) UNIQUE NOT NULL,
  display_order INTEGER DEFAULT 999,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Призначення:**
- Єдине джерело істини для областей
- Визначає порядок відображення (`display_order`)
- Використовується для випадаючого списку при створенні підрозділів

### **3.2. Таблиця `departments` (підрозділи)**
```sql
CREATE TABLE departments (
  department_id SERIAL PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  region_id INTEGER REFERENCES regions(region_id) ON DELETE CASCADE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Зв'язок:**
- `region_id` → FK на `regions.region_id`
- **CASCADE:** при видаленні region → видаляються всі departments
- При видаленні department → CASCADE видаляє майно + приміщення

## 4. ВИМОГИ

### **4.1. Порядок областей (блоків)**

**Послідовність:**
1. Київська (display_order = 1)
2. Харківська (display_order = 2)
3. Полтавська (display_order = 3)
4. Рівненська (display_order = 4)
5. Одеська (display_order = 5)
6. Житомирська (display_order = 6)
7. Решта областей (display_order = 10+) — алфавітно

### **4.2. Структура блоків**

**Спеціальні області (6 шт):**
- Відображаються як **окремі візуальні блоки**
- Усередині блоку підрозділи розміщені **горизонтально**
- Київська — **flex** (підрозділів >10)

**Звичайні області (решта):**
- Відображаються як візуальні блоки
- Усередині — підрозділи (1 або кілька)

### **4.3. Підрозділи в БД**

**Спеціальні підрозділи (створено в seed):**
- «Управління» — в кожній із 6 спеціальних областей
- «ТвУЗ» — в кожній із 6 спеціальних областей
- Київська область: додатково «Підрозділ_1» ... «Підрозділ_10»

**Звичайні підрозділи:**
- Назва = назва області + «Управління» (наприклад, «Львівське Управління»)

**Обов'язкове поле:**
- `region_id` — FK на `regions.region_id`

### **4.4. Будівлі за замовчуванням (створено в seed)**

**Всім підрозділам:**
- Будівля «Управління»

**6 спец-підрозділам (Управління, ТвУЗ):**
- Додатково будівля «ТвУЗ»

**Київські підрозділи (Підрозділ_1 ... _10):**
- Будівля «Будівля_підрозділу»

### **4.5. CRUD для підрозділів (global_admin only)**

**Створити підрозділ:**
- Поля: `name` (назва), `region_id` (область з випадаючого списку)
- Приклад: «Львівське Управління» → region_id = 7 (Львівська)

**Редагувати підрозділ:**
- Можна змінити назву та область
- `region_id` зберігається при зміні назви

**Видалити підрозділ:**
- Cascade: видаляє підрозділ + майно + приміщення
- Підтвердження з попередженням

### **4.6. CRUD для областей (global_admin only)**

**Створити область:**
- Поля: `region_name` (назва), `display_order` (порядок)
- Приклад: «Львівська» → display_order = 10
- Створює **запис в таблиці `regions`**
- Може бути порожньою (без підрозділів)

**Редагувати область:**
- Можна змінити назву та порядок відображення
- Зміни впливають на UI порядок

**Видалити область:**
- Видаляє **запис з `regions`**
- **CASCADE:** видаляє всі підрозділи з цим `region_id`
- Cascade: підрозділи → майно → приміщення
- Підтвердження з попередженням про кількість підрозділів

**Відновлення області після видалення:**
- Створити область заново → вона з'явиться в випадаючому списку
- Створити підрозділи → вони прив'яжуться до області
- Область з'явиться в UI

### **4.7. Видалення приміщень**

**Глобальний адмін:**
- Може видаляти будь-які приміщення в будь-якому підрозділі

**Адмін підрозділу:**
- Може видаляти будь-які приміщення у **своєму підрозділі**

**Видалення будівлі з майном:**
- Майно втрачає прив'язку до будівлі (location_id = NULL)

## 7. ПРИМІТКИ

- **Таблиця `regions`** — єдине джерело істини для областей
- Область існує в БД незалежно від підрозділів
- Видалення області → CASCADE видаляє всі підрозділи
- Створення області → додає запис в `regions`, підрозділи створююються окремо
- Випадаючий список областей → `GET /api/regions`
- Порядок відображення → `display_order` (1-6 для спец, 10+ для решти)
- Київські підрозділи (Підрозділ_1 ... _10) — для тестування flex

## 9. ВИКОРИСТАННЯ API

### **Отримати області (для випадаючого списку):**
```javascript
const regions = await regionsAPI.getAll();
// [{ region_id: 1, region_name: 'Київська', display_order: 1 }, ...]
```

### **Створити підрозділ:**
```javascript
await departmentsAPI.create({
  name: 'Львівське Управління',
  region_id: 7  // Львівська
});
```

### **Створити область:**
```javascript
await regionsAPI.create({
  region_name: 'Львівська',
  display_order: 10
});
```

### **Видалити область (з підрозділами):**
```javascript
await regionsAPI.remove(regionId);
// CASCADE: видаляє область + всі підрозділи + майно + приміщення
```
